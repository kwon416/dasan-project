## KRDS(정부 표준 디자인 시스템) 완전 가이드

KRDS(Korea Government Design System)는 대한민국 정부의 모든 디지털 서비스에서 일관되고 접근 가능한 사용자 경험을 제공하기 위해 설계된 종합 디자인 시스템입니다. 이 시스템은 컴포넌트, 기본 패턴, 서비스 패턴의 세 가지 계층으로 구성되어 있습니다.[^1][^2]

***

## 1. 컴포넌트 계층 (UI Components)

컴포넌트는 UI의 가장 작은 단위로, 과업에 상관없이 일관성 있게 사용되는 공통 요소입니다.[^1]

### 1.1 레이아웃 \& 네비게이션 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **공식 배너(Masthead)** | 사용자가 대한민국 정부 조직에서 운영하는 서비스임을 식별할 수 있도록 제공하는 정보 배너[^1] |
| **운영기관 식별자(Identifier)** | 서비스 운영 주체의 상위 기관을 안내하여 신뢰성을 전달하는 핵심 요소[^1] |
| **헤더(Header)** | 웹사이트의 최상단 영역으로 브랜드 이미지 전달 및 통합검색, 메인 메뉴 등 핵심 탐색 수단 제공[^1] |
| **푸터(Footer)** | 화면의 마지막 요소로 추가 탐색 수단과 유용한 링크 제공[^1] |
| **건너뛰기 링크(Skip link)** | 키보드 사용자를 위해 반복되는 콘텐츠를 건너뛰고 주요 콘텐츠로 빠르게 이동하게 하는 접근성 요소[^1] |

### 1.2 메뉴 \& 탐색 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **메인 메뉴(Main menu)** | 페이지 탐색 시 가장 많이 사용되는 유형의 메뉴로 중요도가 높은 페이지로의 링크 포함[^1] |
| **브레드크럼(Breadcrumb)** | 탐색 계층 구조를 표시하여 현재 위치 파악 및 상위 수준 화면으로의 이동 가능[^1] |
| **사이드 메뉴(Side navigation)** | 서브 화면 내에서의 이동을 위한 좌측 사이드바 형태의 메뉴[^1] |
| **콘텐츠 내 탐색(In-page navigation)** | 본문의 구조를 훑어보고 원하는 콘텐츠로 빠르게 이동하도록 하는 탐색 수단[^1] |
| **페이지네이션(Pagination)** | 많은 양의 콘텐츠를 여러 화면에 나누어 탐색하는 요소[^1] |
| **탭바(Tab bars)** | 화면 하단에 고정되어 핵심 화면으로 빠르게 접근하게 하는 모바일 네비게이션[^1] |
| **링크(Link)** | 다른 서비스, 화면, 섹션으로 이동하는 데 사용되는 탐색 요소[^1] |

### 1.3 콘텐츠 표시 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **구조화 목록(Structured list)** | 유사하거나 관련된 콘텐츠 집합을 표현하여 상세 정보 탐색 및 기능 실행 수단 제공[^1] |
| **긴급 공지(Critical alerts)** | 본문 상단에 강조되어 표시되는 배너로 긴급하거나 중요한 정보 전달[^1] |
| **이미지(Image)** | 단일 이미지 또는 이미지 시퀀스로 텍스트보다 효과적인 정보 전달[^1] |
| **캐러셀(Carousel)** | 하나의 콘텐츠 영역 내에 여러 개의 서로 다른 콘텐츠를 좌/우로 회전시켜 표시[^1] |
| **탭(Tab)** | 상호배타적인 여러 개의 콘텐츠 섹션을 버튼으로 전환[^1] |
| **표(Table)** | 데이터를 행과 열로 조직화하여 많은 양의 정보를 빠르게 확인 및 비교[^1] |
| **텍스트 목록(Text list)** | 계층 구조가 있는 텍스트 블록을 읽기 쉽게 구성[^1] |
| **배지(Badge)** | 컴포넌트 근처에 표시되는 작은 문자 또는 숫자로 상태 정보 표시[^1] |
| **아코디언(Accordion)** | 관련된 여러 콘텐츠 섹션을 수직으로 쌓여 있는 형태로 표현하여 선택적으로 확장/축소[^1] |
| **디스클로저(Disclosure)** | 부가적인 정보를 표시하거나 숨기는 요소로 기본으로 축소된 상태 제공[^1] |

### 1.4 입력 컨트롤 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **버튼(Button)** | 기능이나 동작을 실행하거나 기능을 사용하기 위한 상태로 변경하는 요소[^1] |
| **라디오 버튼(Radio button)** | 여러 개의 옵션 중 한 개의 값만 선택 가능하며 상호배타적[^1] |
| **체크박스(Checkbox)** | 여러 개의 옵션 중 한 개 이상의 값을 선택 가능하며 상호비배타적[^1] |
| **셀렉트(Select)** | 여러 개의 옵션 목록을 팝업으로 제공하여 한 개의 값 선택[^1] |
| **토글 스위치(Toggle switch)** | 상호 배타적인 두 가지 상태를 전환[^1] |
| **텍스트 입력 필드(Text input)** | 사용자가 한 줄의 짧은 텍스트를 입력[^1] |
| **텍스트 영역(Textarea)** | 사용자가 여러 줄의 텍스트를 입력[^1] |
| **날짜 입력 필드(Date input)** | 사용자가 특정 날짜 또는 기간을 입력 또는 선택[^1] |
| **파일 업로드(File upload)** | 디바이스의 로컬 파일을 선택하고 첨부[^1] |
| **태그(Tag)** | 키워드 또는 레이블을 사용하여 콘텐츠 분류[^1] |

### 1.5 피드백 \& 알림 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **모달(Modal)** | 대화창으로 기본 창에 종속되고 가장 상단에 표시되어 단일 과업에 집중[^1] |
| **달력(Calendar)** | 날짜와 관련된 정보와 기능 제공[^1] |
| **스피너(Spinner)** | 페이지나 요소의 다양한 처리 상태를 시각적으로 표시하여 대기 안내[^1] |
| **단계 표시기(Step indicator)** | 사용자가 거쳐야 하는 일련의 단계를 시각화하여 진행 상태 피드백 전달[^1] |
| **스플래시 스크린(Splash screen)** | 모바일 애플리케이션 실행 전에 제공되는 화면[^1] |

### 1.6 도움말 \& 안내 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **도움 패널(Help panel)** | 본문 콘텐츠의 개념/용어 설명, 옵션의 구성, 이용 방법 정보 제공[^1] |
| **따라하기 패널(Tutorial panel)** | 이용 절차에 따라 단계적으로 수행하도록 도와주는 코치마크 실행 패널[^1] |
| **맥락적 도움말(Contextual help)** | 컴포넌트 주변에 배치되어 상태나 상세 정보 제공[^1] |
| **코치마크(Coach mark)** | 새로 도입된 기능을 안내하거나 복잡한 과업을 단계별로 도와주는 고맥락적 도움말[^1] |
| **툴팁(Tooltip)** | 요소에 제공되는 짧은 부가 설명으로 마우스 호버 또는 초점 이동 시 표시[^1] |

### 1.7 접근성 \& 다국어 컴포넌트

| 컴포넌트 | 설명 |
| :-- | :-- |
| **접근 가능한 미디어(Accessible multimedia)** | 미디어 콘텐츠에 설명 제공하는 요소[^1] |
| **숨긴 콘텐츠(Visually hidden)** | 화면에는 보이지 않지만 스크린 리더 사용자는 읽을 수 있도록 설계[^1] |
| **언어 변경(Language switcher)** | 서비스 콘텐츠 표시 언어 변경 또는 외국어 서비스로 이동[^1] |
| **화면 크기 조정(Resize)** | 텍스트를 포함한 화면 정보의 확대 또는 축소[^1] |

## 2. 기본 패턴 계층 (Global Patterns)

기본 패턴은 컴포넌트 요소들이 조합되어 핵심 과업을 반복적으로 함께 사용되는 UI 집합입니다.[^2]


| 패턴 | 설명 | 주요 용도 |
| :-- | :-- | :-- |
| **개인 식별 정보 입력** | 특정 개인의 신원을 밝히거나 기본 정보 확인에 사용[^2] | 회원가입, 신청서 작성 |
| **도움** | 서비스 이용 방법, UI 작동 방식, 과업 플로우 관련 정보 제공[^2] | 온보딩, 튜토리얼, 문제 해결 |
| **동의** | 이용 조건, 절차 등의 내용을 읽고 동의 여부 확인[^2] | 약관 동의, 개인정보 처리 동의 |
| **목록 탐색** | 의미적으로 관련 있는 다수 데이터를 연속적으로 배열한 색인[^2] | 검색 결과, 데이터 조회 |
| **사용자 피드백** | 페이지나 기능 이용 경험에 대한 평가 의견 수집[^2] | 만족도 조사, 개선 제안 |
| **상세 정보 확인** | 특정 주제의 정보를 텍스트, 시각적 요소로 표현[^2] | 공지사항, 정책 상세 정보 |
| **오류** | 사용자 요청한 작업을 시스템이 완료하지 못할 때 정보 제공[^2] | 입력 오류, 시스템 오류 메시지 |
| **입력폼** | 사용자가 데이터를 입력하여 서버로 전송[^2] | 신청, 예약, 회원가입 |
| **첨부 파일** | 게시물, 본문에 제공되는 다운로드 가능한 콘텐츠[^2] | 문서 다운로드, 자료실 |
| **필터링·정렬** | 데이터 집합에서 원하는 속성/범주 데이터 항목을 선별 및 조직화[^2] | 검색 필터, 리스트 정렬 |
| **확인** | 사용자 요청 행위가 쉽게 취소할 수 없는 결과로 이어질 때 사용[^2] | 삭제 확인, 결제 승인 |

## 3. 서비스 패턴 계층 (Service Patterns)

서비스 패턴은 디지털 정부서비스의 핵심 과업에 대한 사용자 여정 기반의 UX 설계 가이드입니다.


| 서비스 패턴 | 설명 | 적용 시나리오 |
| :-- | :-- | :-- |
| **방문** | 사용자가 서비스에 접속하여 원하는 상세 정보로 연결되는 과정 | 포털 메인 화면, 서비스 홈페이지 |
| **검색** | 사용자가 큰 데이터 집합에서 원하는 정보를 찾을 수 있도록 도움 | 통합 검색, 맞춤 검색 |
| **로그인** | 사용자의 신원을 확인하는 과정으로 개인화된 경험 제공 | 회원 인증, I-PIN 인증 |
| **신청** | 사용자가 요구 사항을 알리기 위해 서식을 작성 | 민원 신청, 서비스 신청 |
| **정책정보 확인** | 정부/기관의 행동 방침, 계획, 법률에 관한 정보 확인 | 정책 조회, 법령 검색 |

## 4. KRDS의 핵심 설계 원칙

KRDS는 다음의 4가지 설계 원칙을 기반으로 합니다:

1. **일관성(Consistency)**: 모든 정부 디지털 서비스에서 동일한 컴포넌트와 패턴을 사용하여 예측 가능한 경험 제공
2. **접근성(Accessibility)**: 스크린 리더 사용자, 키보드 네비게이션 사용자 등 모든 사용자를 고려한 설계
3. **사용성(Usability)**: 직관적인 인터페이스와 명확한 정보 구조로 사용자가 과업을 쉽게 완료
4. **플랫폼 고려(Platform Considerations)**: 데스크톱, 태블릿, 모바일 등 다양한 기기 환경 대응

***

## 5. KRDS 적용 수준

각 서비스 패턴에 대해 다음의 3단계 적용 수준을 제시합니다:

- **필수(Do)**: 모든 정부 디지털 서비스가 반드시 따라야 할 최소 기준
- **권장(Better)**: 사용자 경험 향상을 위해 권장하는 수준
- **우수(Best)**: 탁월한 사용자 경험을 제공하는 모범 사례

***

KRDS는 정부 디지털 서비스의 표준화된 UI/UX 경험을 제공함으로써 국민 모두가 쉽고 일관되게 공공 서비스를 이용할 수 있도록 설계된 종합적인 디자인 시스템입니다.
